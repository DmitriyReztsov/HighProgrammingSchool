3. пользовательские метафоры (что именно пользователь/заказчик понимает под тем, что в рамках проекта 
он называет, например, "Автомобиль" или "Товар" или "Клиент")
- Пользователь - активный игрок
- комбинация три-в-ряд - состояние системы, при которой в трех связанных между собой элементах (объектах,
таблицы или узлах графа) содержатся одинаковые значения поля. Элементы связаны либо по горизонтали, либо
по вертикали (связи для матрицы - отличие индексов в списках на 1 на одной позиции, для графов - движение
по ребрам одного типа)
- перестановка элементов. Изменение состояния системы при которой объекты указанных пользователем ячеек
взаимно меняют свои значения. Ячейки должны находиться в связи по горизонтали или вертикали
- удаление элементов. Изменение состояния системы, когда ячейки, удобвлетворившие условию "три-в-ряд"
меняют свои значения на значения ячеек, связанных с ними вверх по веритикали рекурсивно до ячейки, 
у которой отсутствует такая связь (1й ряд). Такая чейка заполняется случайно сгенерированным значением.
- ход. Ввод игрока такой комбинации из двух связанных по горизонтали или веритикали ячеек, смена значений
в которых приведет к ситуации "три-в-ряд"
- координаты ячейки. Смещение по графу или матрицы относительно первого элемента. Первое значение - по
горизонтали (обозначение столбцов), второе - по вертикали (строки)
