1. Создание объектов.
1.1. список комбинаций создаем через метод Window._create_combination_list().
1.1.1. Объекты класса Combination создаем через фабричный метод, по одному объекту для каждого класса-потомка.
1.1.2. Параллельно создаются объекты класса Bonus для обслуживания объектов комбинаций. Так же по экземпляру для потомка. Присоединяются к комибанициям по заданному мэппингу.

1.2. Объект Gamer создаем через стандартный конструктор.

1.3. Объект счета Score для игрока создем через фабричный метод.

1.4. Объект шага Step создается через фабричный метод.
4.1. По координатам, введенным игроком, находим объекты ячеек. На их основе создаем объект шага.

1.5. Объекты ячеек Cell игрового поля PlayGround создаются через фабричный метод PlayGround.generate_playground().
5.1. Ячейка создается через свой фабричный метод, заполняя значение случайным образом.

2. Обработка событий.
2.1. Начало игры
- создание объекта игрока Gamer,
- объекта окна Window, далее в нем
- объектов комбинаций Combination с бонусами Bonus,
- объекта счета игрока Score
- генерация объекта игрового поля PlayGraound, далее в нем
- создание объектов ячеек игрового поля Cell
- приглашение к вводу хода  

2.2. Ввод игроком хода
- создание объекта шага Step,
- обработка хода замена ячеек друг с другом
- сохранение состояния поля и шага в стеке шагов StepStack

2.3. Проверка поля на комбинации
2.3.1. комбинация Combination найдена
- применение бонуса Bonus
- начисление очков Score
- отображение нового значения очков, отображение нового состояния поля
- повторить шаг
2.3.2. комбинация не найдена как результат хода
- конец игры
2.3.3. комбинация не найдена как результат автоматического пересчета поля
- конец хода, ожидание ввода нового шага

3. Тесты для типовых сценариев
3.1. сформировать поле с заданными комбинациями и проанализировать его:
- линия горизонтальная 2 элемента - результат: нет комбинаций
- линия вертикальная 2 элемента - нет комбинации
- линия горизонтальная 3 элемента - есть комбинация
- линия вертикальная 3 элемента - есть комбинация
- линия горизонтальная 4 элемента - есть комбинация
- линия вертикальная 4 элемента - есть комбинация
- крест - есть комбинация
3.2. сформировать поле с заданными комбинациями и применить к нему реакцию бонуса. Ожидаемый резутат - новое поле, ячейки с комбинациями заменены на вышестоящие или новые.
3.3. проверить корректность ввода шага:
- ввести сущестсующие ячейки - должен сформироваться объект Step
- ввести одну несуществующую ячейку - объект Step не должен сформироваться, пользователю должно вернуться сообщение о недопустимости координат
- ввести обе несуществующие ячейки - объект Step не должен сформироваться, пользователю вернуться сообщение о недопустимости координат
3.4. проверить корректность выхода из игры:
- сформировать игровое поле без комбинаций после шага пользователя - выход из игры
- сформировать игровое поле без комбинацией, без пользователя - возврат на этап ввода хода
3.5. интеграционное тестирование под контролем формирования поля. Ввод шага - анализ поля - результат
