0. Модуль-диспетчер. Собственно игровая логиа. Прием ввода, обработка хода, обработка поля, вывод нового поля.
1. Модуль игрового поля. Содержит структуру клеток. Состоит из клеток.
    1.1. Структура игрового поля. Либо граф, либо обертка над списком списков.
    1.2. Клетка. Хранит значение. Состоит из значения.
        1.1.1. Значение. В текущих рамках ТЗ - обертка надо примитивом str. Имеет значение, может сравнивать значения,
            обеспечивать правильное значение, генерировать новое значение.
2. Модуль пользовательского ввода. Меняет значения в клетках.
3. Модуль пользовательского вывода. Отрисовывает игровое поле.
4. Модуль расчетов.
