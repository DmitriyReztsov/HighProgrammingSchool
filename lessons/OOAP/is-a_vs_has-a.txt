class Document и class DocumentProject (который должен конкретизировать документы для опредлеенного проекта). 
Казалось бы, что можно сделать так: class DocumentProject(Document), расширить класс документов какими-то аттрибутами
и методами, характерными для работы с проектами, но правильнее было бы включить документ как поле класса DocumentProject
вместе с полем проекта.

Например, есть класс Проекта, описывающий общие параметры проекта (адрес, участники, суммы). Мы хотим ввести класс
для программы кредитования на основе проекта. Было бы неправильным наследоваться от класса Проекта, хотя для кредитования
нам нужны все те же поля, плюс несколько полей, описывающих кредит. Если, например, вместо проекта мы будем кредитовать
на основе заключенного контракта, который будет содержать примерно те же поля, что и проект (стороны контракта, суммы),
но по сути являться другим типом данных, то нам пришлось бы создавать отдельный класс, выполняющий ту же роль.
В данном случае, подошло бы отношение has-a.
